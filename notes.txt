الحالة 1️⃣: أبغى أشوف النسخة القديمة بس (بدون ما ألمس الشغل الحالي)
git checkout 1234567



لو خلصت وتبي ترجع لأحدث نسخة (مثلاً الفرع master أو main):
git checkout master
او
git checkout main


الحالة 2️⃣ (المفضلة): أريد أبدأ تعديلات جديدة انطلاقًا من نسخة قديمة
الأفضل تسوي فرع جديد من النسخة القديمة وتشتغل عليه، بدون ما تخرب تاريخ الفرع الأساسي.
الفكرة:

عندك مثلاً الفرع الأساسي: master

عندك نسخة قديمة برقم: 1234567

بنسوي فرع جديد يبدأ من النسخة القديمة مثلاً اسمه: old-fix

git switch -c old-fix 1234567
لو الأمر switch ما اشتغل، استخدم:
git checkout -b old-fix 1234567


old-fix = اسم الفرع الجديد (تقدر تسميه أي شيء يناسبك).

بعد هالأمر:

الملفات في المجلد تصير مثل ما كانت في النسخة القديمة.

وتكون الآن واقف على فرع جديد مستقل.

الآن اشتغل عادي:

عدّل الملفات.

بعد كل مجموعة تعديلات:
git add .
git commit -m "تعديل انطلاقًا من النسخة القديمة"


لو تبغى ترجع للفرع الأساسي:

bash
Copy code
git switch master
# أو
git checkout master

وتقدر تنتقل بين الفرعين متى ما حبيت:

git switch old-fix
git switch master



الحالة 3️⃣ (خطِرة): أريد الفرع الرئيسي نفسه يرجع لنسخة قديمة

و الفرع عندك اسمه master وتبغاه يرجع للنسخة 1234567:
تأكد إنك على نفس الفرع:

git checkout master


نفّذ:

git reset --hard 1234567


هذا يسوي:

يحرك مؤشر الفرع master إلى النسخة القديمة.

يعدّل الملفات في المجلد لتطابق تلك النسخة.

أي commits بعدها تختفي من تاريخ الفرع.

ما أنصحك تبدأ بهالطريقة الآن، الأفضل تجرب الحالة 2 (فرع جديد من النسخة القديمة) لين ترتاح مع Git.
